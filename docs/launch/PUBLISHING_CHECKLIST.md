# Publishing checklist

| Platform | Automatic publication | Current state | Manual next step | Recommended asset | URL / result |
| --- | --- | --- | --- | --- | --- |
| GitHub repository | Yes | Completed | Maintain README and release | `01-main-light.png` + GIF | Repository homepage |
| GitHub Discussions | Yes | Completed | Reply to feedback | Current formal set | Discussions 1–5 |
| GitHub Issues | Yes | Completed | Triage reproducible contributions | None required | Issues 6–9 |
| GitHub Release | Yes | Pending rebuilt artifacts | Publish after CI/build verification | Release notes | v1.5.2 |
| GitHub Pages | Permission dependent | Pending | Enable Pages from `main` / `docs` if desired | Main screenshot | — |
| Reddit | No | Draft ready | Log in, read current rules, tailor one post | Main screenshot | See `reddit-launch.md` |
| Hacker News | No | Draft ready | Submit Show HN from a human account | GIF / source link | See `hackernews-launch.md` |
| Product Hunt | No | Draft ready | Use a personal maker account | Gallery set | See `producthunt-final.md` |
| AlternativeTo | No | Draft ready | Submit through platform UI | Main screenshot | `alternativeto-final.md` |
| DEV Community | No | Draft ready | Publish technical article manually | Formula/table evidence | `dev-community-final.md` |
| Peerlist | No | Draft ready | Publish manually | Main screenshot | `peerlist-final.md` |
| Indie Hackers | No | Draft ready | Publish manually | Main screenshot | `indiehackers-final.md` |
| X / LinkedIn | No | Draft ready | Review and post manually | GIF / main screenshot | `x-launch-final.md`, `linkedin-final.md` |

Never automate posting to external communities without the account holder's
explicit platform-level authorization and a final review of live rules.
