# Community and Awesome List preparation

Check each repository's contribution rules and scope before opening an issue or
pull request. A listing is appropriate only if its maintainers accept software
that is currently released, open source, and relevant to the list.

## One-line listing

**CleanText Studio** — Privacy-first, local-first Windows application for
cleaning copied/AI-assisted text, preserving structure and supported formulas,
and exporting DOCX/TXT.

## Community route

1. Start on GitHub: publish a complete README, current release, issue forms,
   discussions, and a concise technical announcement.
2. Post an audience-specific technical introduction to one community at a time.
3. Gather reproducible feedback before approaching curated Awesome Lists.
4. Submit a small, list-format-compliant PR with the one-line description.
5. Continue maintenance after any listing; do not treat a list as a one-time
   promotional channel.

## Communities to evaluate before posting

- r/opensource — open-source project feedback.
- r/LocalLLaMA — local-first/AI workflow discussion, with strict promotion rules.
- r/artificial — AI product and workflow context.
- r/Python — only under the community's current project-sharing policy.
- Hacker News `Show HN` — technical maker discussion.
- Product Hunt — launched product and maker story.

Avoid posting a generic project announcement to communities that prohibit
self-promotion, including r/programming unless its live rules explicitly say
otherwise.
