# README content specification

Every official README page must describe the same released product using its
own natural language. Product names, commands, filenames, URLs, and technical
identifiers remain literal.

## Required ordered topics

1. Hero, positioning, language navigation, badges, and release download.
2. Why CleanText Studio and a safe before/after example.
3. Core cleanup, local-first privacy, tables, supported equations, installation,
   quick start, cleaning modes, export formats, and optional AI refinement.
4. Formal v1.5.2 screenshots, limitations, technical overview, development
   setup, tests, roadmap, contributing, security, license, acknowledgements,
   and a natural invitation to support or contribute.

## Screenshot policy

All pages reference the formal `assets/screenshots/v1.5.2/` set. Marketing
pages must not use formula-index images, historical screenshot directories, or
images showing raw backslash commands. The five primary assets are
`01-main-light.png`, `02-main-dark.png`, `03-before-after-light.png`,
`05-table-export.png`, and `06-word-export.png`.

## Formula wording

Use this statement or an accurate localized equivalent:

> Supported formulas can be exported as editable Word equations; unsupported or malformed expressions may require review.
